## Profile Dependencies Matrix

| Profile | Depends On | Services Count |
|---------|-----------|----------------|
| core | (none) | 1 (mongo) |
| ui | core | 4 services |
| ml-ops | core | 2 services |
| explainability | core + ml-ops | 1 service |
| fairness | core + ml-ops | 1 service |
| privacy | core | 1 service |
| llm | core | 2 services |
| security | (standalone) | 3 services |
| storage | (standalone) | 1 service |

## Resource Scaling

```
Minimal:  ui                           →  5 containers  →  3GB RAM
Light:    ui + storage                 →  6 containers  →  3.5GB RAM
Medium:   ui + ml-ops + explainability →  8 containers  →  5GB RAM
Heavy:    ui + ml + llm                → 11 containers  →  8GB RAM
Maximum:  all profiles                 → 16 containers  → 12GB RAM
```

## Communication Patterns

1. **UI → Backend Services**: HTTP/REST APIs
2. **Services → MongoDB**: MongoDB wire protocol (27017)
3. **Moderation Layer → Model**: Internal HTTP
4. **Services → Reporting**: Internal HTTP for PDF generation
5. **Frontend → User**: HTTP (ports 30010, 30055)

## Security Considerations

- All services run in isolated `rai-network`
- MongoDB accessible on localhost only by default
- Services communicate using container names (not localhost)
- Volumes persist data independently of containers
- Health checks ensure dependencies are ready
